## 考试背景

你将基于我们提供的初始代码，完成一个 **「个人记账本」** 的开发，包括命令行客户端和数据同步服务端。

项目已经搭好了基本骨架（数据模型、存储层、业务层、命令行入口），但代码中存在若干问题需要你修复，同时还有待实现的功能模块。

初始数据文件 `data/data.csv` 包含了一批历史交易记录，你的程序需要能够读取并加载这些数据。

## 任务一：修复初始项目（建议 30~45 分钟）

初始项目存在以下类型的问题：

1. **编译/语法错误**：部分代码有拼写错误或缺少导入，导致无法正常构建
2. **数据模型问题**：实体定义或数据结构存在设计缺陷
3. **逻辑 Bug**：部分已实现的方法存在逻辑错误
4. **架构问题**：某些代码的分层或依赖关系不合理

**你的任务**：

- [ ] 让项目能够成功编译/运行
- [ ] 找出并修复所有你能发现的 Bug 和设计问题
- [ ] 在代码中用注释标注你修复了什么、为什么（格式：`// FIX: 描述`）

## 任务二：实现核心功能（建议 1~1.5 小时）

在修复好的项目基础上，完成以下功能。数据以 JSON 文件（或你选定的格式）持久化存储，读写时需要考虑**文件锁**以防止并发冲突。

### 2.1 数据导入与导出

- 从 `data/data.csv` 读取历史交易记录并加载到系统中，对异常数据进行合理处理
- 实现 `exportToCSV(filePath)` 导出功能，表头：日期, 类型, 金额, 分类, 备注

### 2.2 增删改查

- 支持添加收入/支出记录（日期、类型、金额、分类、备注）
- 支持排序（金额降序或你选择的一种排序字段）排序算法必须自己实现，比如冒泡算法。

### 2.3 统计功能

- **月度统计**：指定年月，返回总收入、总支出、净收入（金额计算必须精确）
- **分类统计**：指定年月，返回各支出分类的金额和占比，按金额降序

### 2.4 命令行交互

补充实现 CLI 命令：`add`、`list`、`summary`、`category`、`delete`、`export`、`help`、`quit`。

输入错误时应给出友好提示，程序不应崩溃。

## 任务三：用户认证与数据加密（建议 30~45 分钟）

### 3.1 口令登录

为记账本 CLI 增加用户登录功能：

- 首次运行时要求设置用户名和口令
- 后续每次启动需要输入用户名 + 口令才能进入系统
- **口令存储要求**：使用加密算法（自行设计加密方案）的方式存储口令，禁止明文存储
- 口令验证失败时给出友好提示，连续失败 3 次后退出程序
- 支持口令的修改功能，修改时需要验证旧口令

### 3.2 数据文件加密

设计并实现一个加密系统，保护记账数据文件。

**基本要求**：
- 使用 **AES-256** 算法对数据文件进行加密，并在注释中说明你的加密方案的设计的原因
- 程序读取数据时自动解密，写入时自动加密

**密钥管理要求**：
- 不得将加密密钥硬编码或明文存储
- 使用 **PBKDF2** 从用户口令派生加密密钥，迭代次数不低于 10000 次
- 每次加密使用**随机生成的 IV（初始向量）**，IV 与密文一起存储

**完整性校验**（可选，加分项）：
- 如使用 CBC 模式，需额外实现 HMAC 校验防止密文被篡改（先加密后MAC）
- 如使用 GCM 模式，已自带认证，请说明 GCM 的认证机制

在代码注释或 README 中说明：你选择的加密模式、密钥派生方案、IV 处理方式，以及为什么这样设计。

## 任务四：数据同步服务端（建议 30~45 分钟）

实现一个 HTTP 服务端，支持记账数据的远程同步。

### 需求

1. 服务端可以使用**任意编程语言和任意框架**（不限于初始项目所选语言）
2. 不要求实现 Server 的 UI 界面，**命令行启动即可**
3. 实现以下 API（RESTful 风格建议）：
   - 上传本地数据到服务端（全量或增量，自行设计）
   - 从服务端拉取数据到本地
   - 查询服务端当前数据状态（记录数、最后更新时间等）
   - 考虑到时间有限，允许不使用数据库，直接模拟存储在内存或文件中，但需要设计合理的数据结构和接口

## 面试讨论预告

提交后的视频面试中，我们会围绕以下方向讨论：

1. **修复思路**：你是怎么定位 Bug 的？排查过程是什么？
2. **数据读取**：data.csv 中有哪些数据？你是怎么处理的？
3. **排序实现**：你的排序是怎么做的？不同字段的排序有什么区别？
4. **金额精度**：你是怎么处理金额计算的？为什么？
5. **安全设计**：口令如何加密的，你是如何设计的？
6. **文件锁**：多进程同时读写数据文件会怎样？你怎么处理的？
7. **同步策略**：服务端和客户端数据冲突时怎么办？
8. **设计决策**：你的代码为什么这样组织？
9. **没做完的部分**：如果再给你 2 小时会怎么做？
